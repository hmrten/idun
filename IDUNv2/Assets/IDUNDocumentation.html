IDUN

Documentation

Introduction

IDUN is a software project made by Mårten Hansson and Nikola Vorkapic.

IDUN is a Universal Windows Application software that enables sensor data collection from a Rasberry Pi 3 device, reporting pre-set triggers to IFS Cloud, together with Maintenance requests and Fault reports, and on-screen information display across multiple Windows platforms such as Desktop PC, Tablets, Mobile phones and other IoT Devices.

Goal of this project is to increase awareness of the endless possibilities of multi platform solutions, IoT devices (such as sensor measurements)  and Cloud (in this particular case IFC cloud) usage for Data Storage and effortless communication between any number of devices and its’ users.

Requirements

Requirements for fully operational IDUN are:

Raspberry Pi 3

Sense HAT

Raspberry Pi 7” Touchscreen Display (Window Resolution 800 x 480 pixels)

Internet Connection

Windows IoT Core installed

IDUN is multi platform compatible with any platform running Windows 10. Sensor readings will not be possible on any other platform beside Raspberry Pi 3.

Available only in English language.

Technical Details

IDUN is based on Universal Windows Platform (UWP) software architecture.

IDUN is Metro-style application and can be run on Windows 10, Windows 10 Mobile and Windows 10 IoT Core without need for separate installation files.

Universal Windows Applications cannot run on previous versions of Windows. Windows 10 is the minimum requirement for any UWP application.

For more information on UWP please visit:

https://msdn.microsoft.com/windows/uwp/get-started/whats-a-uwp

IDUN uses Raspberry Pi 3 and Sense HAT for sensor data input. Data is being read from the calibrated HTS221, LPS25H and LSM9DS1 sensors which return scalar values for temperature, pressure, humidity and vector values for acceleration, magnetic field and rotational motion (gyroscope).

Raspberry Pi 3 Hardware specification:

https://www.raspberrypi.org/documentation/hardware/raspberrypi/README.md

Sense HAT technical documentation:

https://www.raspberrypi.org/documentation/hardware/sense-hat/

Sense HAT hardware schematics:

https://www.raspberrypi.org/documentation/hardware/sense-hat/images/Sense-HAT-V1_0.pdf

Sensor data is being displayed on the IDUN UI and saved locally for further analysis.

IDUN has option to set up Sensor Triggers. Sensor Triggers are defined by Sensor Type, Operator, Value and Template.

When a certain Sensor Value is Above or Below predefined Value it will run Template which can set off different warnings and requests directly in the IFS database.

Templates base is consisted of Directive, Fault Description, Discovery method, Symptoms and Priority.

For a device, running IDUN, to be able to send data to IFS cloud, authorisation is required.

Device and its’ user need to be identified and connected through Cloud URL and SystemID.

IDUN is currently using Addovation Cloud assemblies and IFS Demo Cloud.

Other assemblies being used are:

NETCore UWP

Newtonsoft Json

SQLite

IoT Extensions for the UWO

Visual C++2015 Runtime

Xamarin

Xaml

IDUN offers control over Sense HAT LED grid through graphical interface and Text-to-Speech control using Windows pre-installed voice packages.

Data saved locally is being saved in several different ways depending on permanency requirements. Data that is required only temporarily is being saved in Windows Local Settings and Local Files through Json Serializer and for more permanent solution Database is being used through SQLite controls.

Data saved remotely is controlled by IFS services.
